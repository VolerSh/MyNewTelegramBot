Детализация: Модуль парсинга (Scrapy)

← Назад к общей структуре

→ На входе

Модуль получает команду на запуск от telegram_bot/handlers.py в виде вызова функции run_spider() с поисковым запросом (например, "lenovo thinkbook").

↓ Что делает

1. Запуск (utils.py): Функция run_spider() программно конфигурирует и запускает процесс Scrapy.

2. Парсинг (spiders/yandex_market.py): "Паук" Scrapy отправляет HTTP-запросы на Яндекс.Маркет, следует по страницам пагинации и извлекает из HTML-кода "сырые" данные: название, цену, ссылку.

3. Декомпозиция (decomposer.py): Каждый полученный товар обрабатывается классом LaptopDecomposer, который "разбирает" строку с названием на структурированные технические характеристики (процессор, память, накопитель и т.д.).

← На выходе

Модуль возвращает список структурированных данных (словарей Python), готовых для сохранения. Этот список передается в модуль базы данных для сохранения в marketplace_deals.db.
